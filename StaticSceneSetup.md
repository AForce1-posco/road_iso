# 정적 씬 셋업 (road_iso, Unity 2020.3)

## 씬 만들기 — 1분 컷
1. `File > New Scene` → `Assets/Scenes/01_StaticScene.unity`으로 저장
2. 빈 GameObject 생성 (이름 아무거나, 예: `Bootstrap`)
3. `StaticSceneSetup` 컴포넌트 추가
4. Play — 끝. 카메라·조명·트레이·대시보드 전부 자동 생성

기존 씬의 Main Camera / Directional Light가 있으면 그대로 쓰고, 없으면 만들어 줌.

## 조작
| 입력 | 동작 |
|---|---|
| 좌클릭 | 배치 |
| 우클릭 | 제거 |
| R | 요(yaw) 90° 회전 |
| T | 눕히기(pitch) 90° |
| F | 커서 아래 화물 고정↔자유 |
| 하단 카드 | 화물 종류 선택 (마우스 휠로 가로 스크롤) |
| 물리 버튼 | ON=자유 화물 중력 시뮬 (파랗게 켜짐) |

## 화물 종류
- 인스펙터의 `Cargo Types` 배열을 **비워두면** `CargoCatalog`(실측 CSV 17종)이 자동 적용됨
- 실측표: `cargo_input_template.csv` (박스 7, 톤백 1, 코일 1, 파이프 번들 4, 배관 파이프 3, 납벨트 1)
- 번들 파이프는 묶음 1개 = 화물 1단위 (질량·단면 = 번들 전체 값)
- 카드의 "재고 n"은 실제 목업 수량 안내 — 배치 개수 제한은 없음

## 형상 표현
- **코일(C-001)**: 절차 생성 도넛 메시 (외경 10 · 내경 4 · 높이 2 cm). 구멍은 시각 표현이고
  콜라이더는 convex라 물리적으로는 막혀 있음 (내경 4cm에 끼일 화물이 없어 MVP에선 무영향)
- **톤백(T-001)**: 캡슐 비주얼 + 박스 콜라이더 (연질 변형은 강체 근사)

## 저장
- `저장` → `persistentDataPath/staging_export.json` (동적 씬 로드용, 스키마 CargoRiskSimulation과 동일)
- `케이스저장` → `Assets/Data/Cases/case_타임스탬프.json` (배치 자동화용 누적)

## 알려진 제약
- S-002(70cm)·S-003(79.5cm) 파이프는 트레이(64×24cm)보다 길어서 안에 안 들어감 —
  배치하면 벽에 걸쳐 얹히는 형태가 됨 (실물 목업도 동일한 물리 조건)
