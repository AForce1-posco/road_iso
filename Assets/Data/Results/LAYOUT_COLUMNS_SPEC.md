> ⚠️ **커버리지 경고 (반드시 명시)**: 이 9개 range CSV는 **case080~500 (=421개)만** 덮습니다.
> - **case001~079 없음** (이 배치 시리즈에 안 들어감)
> - **`401-450` 파일은 절반**(case480~500, 21개만) — 끊긴 게 아니라 원래 케이스 수가 적음
> - ⇒ **"500개 전부 처리"가 아님.** 발표/보고 시 이 커버리지를 밝히지 않으면 공격받음.
> - ⚠️ **파일명 ≠ 내용**: `001-050.csv` 안에는 case001~050이 아니라 **case080~129** (파일명이 배치 내 실행순번이라 +79 오프셋). **반드시 `LayoutID` 컬럼으로 조인.**

기존 주행 시계열 CSV 각 행 **끝에 7개 컬럼**을 덧붙인 파일이 `*_layout.csv`.
생성: **`augment_layout.py`** (같은 폴더, `python3 Assets/Data/Results/augment_layout.py`). 조인 키 = `LayoutID`(=caseNNN) → `Assets/Data/Cases/caseNNN.json`.

## 추가 7개 컬럼 (실축 SI, 트레이 로컬)
| 컬럼 | 내용 | 출처 |
|---|---|---|
| `CogX, CogY, CogZ` | 적재물 무게중심 (m, 트레이 중심 원점 / y는 바닥 기준) | JSON `cog` (생성기 계산값) |
| `MaxHeightM` | 적재 최고 높이 (m) | JSON `maxHeight` |
| `InertiaXX/YY/ZZ` | 관성텐서 대각 (kg·m², **적재물 CoG 기준·트레이 축**) | cargo[]+카탈로그 계산 |

- 배치값은 **정적 상수** → 한 주행(케이스)의 모든 행에서 동일 값 반복(정상, 버그 아님).
- `CargoCount·TotalMassKg`는 기존 컬럼에 이미 있어 중복 추가 안 함.
- ⚠️ **좌표계 주의**: `CogY`는 현재 **bedAnchor 기준(원 JSON y)**. RL 관측·슬라이드·목업은 **바닥면(floorTop) 기준**이라 **~1cm 차이** → 예측기/발표에서 기준 통일 필요 (해소 시 `−floorTopY`).

## 관성모멘트 식
각 화물 = 균일밀도 직육면체. 자체 관성 `Ixx=m/12·(dy²+dz²)` 등 → 회전(Unity Euler) 정렬 `R·I·Rᵀ` → 평행축정리로 적재물 CoG 이동 → 합산. 대각성분 3개 보고.
