# 정적 배치 최적화 강화학습 — 설계 문서 (MDP · 규제 · 보상)

> 대상: road_iso 정적 씬(CargoPlacer + LoadCalculator)에서, 주어진 화물 목록을
> **전복 위험 최소 + 규제 만족 + 공간 효율**로 배치하는 RL 에이전트.
> 1차 보상은 **정적 지표**(LoadCalculator) 기반 — 동적 파이프라인 없이 착수 가능.
> 검증 단계에서 동적 주행(2000 케이스 파이프라인)으로 실제 확인.

---

## 1. 문제 정의 (한 줄)

**입력**: 실어야 할 화물 목록(manifest, 예: 코일2·박스4·파이프3)
**출력**: 각 화물의 트레이 내 위치·회전 (전부 고정 가정)
**목표**: 규제를 지키면서 **안정성(StabilityMargin↑, |LTR|↓, CoG 낮게·중앙)** 최대화, 공간 밀집.

---

## 2. 환경 상수 (프로젝트 실측)

| 항목 | 값 (확정) |
|---|---|
| **적재함(loading area)** | **62 × 21 × 27 cm** (문서 확정) |
| **높이 한도** | **27 cm** 까지 적층 허용 (벽 6cm 아님 — 법적/구조 한도) |
| 바닥 top(화물 기준면) | y = 0.01 m |
| **격자 셀** | **4 cm** → 약 **15 × 5 ≈ 75 셀** (62/4, 21/4). 관측 점유맵도 동일 |
| **적재 한도(payload)** | **7 kg 목업** = 700 kg 실차 (massScale 100). H1 기준 |
| 화물 종류 | 카탈로그 17종 (박스/코일/톤백/파이프/납벨트), **전부 Bounding Box로 추상화** |
| 좌표계 | 설계문서 x=주행·y=좌우·z=높이 / **Unity 시뮬 x=좌우·y=높이·z=주행** → 구현 시 매핑 |
| 지표 소스 | `LoadCalculator` (ComputeCoG / ComputeLoads / ComputeLTR / StabilityMargin / LateralTipG / SupportPolygon) |

---

## 3. MDP 정의

에피소드 = 화물 목록 하나를 처음부터 끝까지 배치하는 1회.

### 3.1 상태 (Observation)
| 그룹 | 내용 | 차원(예) |
|---|---|---|
| 점유 높이맵 | 각 격자셀의 현재 적재 높이(정규화, 0~27cm) | 15×5 ≈ 75 |
| 현재 CoG | (x, z, 높이) 정규화 | 3 |
| 질량 | 현재 총질량 / payload | 1 |
| 안정 지표 | StabilityMargin, |LTR| (현재까지) | 2 |
| 다음 화물 | 크기(w,d,h)·질량·shape(one-hot) | ~8 |
| 남은 목록 | 종류별 남은 개수 벡터 | 17 |

→ 총 관측 ~127차원 (격자 해상도로 조정).

### 3.2 행동 (Action) — 순차 배치, **화물도 에이전트가 선택** ★
매 스텝 **"남은 화물 중 무엇을 · 어디에 · 어느 회전으로 놓을지"** 결정.
- **화물 선택** (이산): 남은 화물 종류/슬롯 인덱스
- **격자 셀 인덱스** (이산): 약 75셀 (4cm 격자) 중 하나
- **회전** (이산): 0° / 90° (박스·파이프)

→ 이산 행동 = (item, cell, rot). ML-Agents `ActionSpec.MakeDiscrete(numItems, ~75, 2)`.
→ 화물 선택까지 학습하므로 **"무거운 걸 먼저·아래에"** 같은 적재 순서 전략도 스스로 익힘.

### 3.3 액션 마스킹 (하드 규제 사전 차단)
`WriteDiscreteActionMask`로 **불가능/불법 셀을 아예 못 고르게**:
- 트레이 밖으로 튀어나오는 셀 (긴 파이프 오버행은 예외 허용)
- 이미 점유된 셀(겹침)
- 받침 없는 2층 위치(공중)
- 배치 시 **과적(payload 초과) 확정**되는 경우
- **이미 다 쓴 화물**(남은 수 0)은 선택 못 하게 마스킹

→ 에이전트는 **유효 (화물,셀,회전) 조합만** 탐색 → 학습 효율↑.

### 3.4 종료 (Termination)
- 화물 목록 다 놓음 → 정상 종료 + **최종 보상**
- 놓을 유효 셀이 없음(꽉 참/규제) → 종료
- (하드 위반이 마스킹을 못 막은 예외) → 큰 감점 + 종료

## 4. 규제 (RuleChecker) — 확정본

> RuleChecker = `LoadCalculator` + 기하검사(AABB)를 모은 **단일 판정 모듈** (RL·검증·정적 대시보드 공유).
> 원칙: **불가능/절대금지 = Hard(마스킹)**, **좋고 나쁨의 정도 = Soft(보상)**, **최소선+정도 = Hybrid**.

### 4.1 Hard Constraint (Action Masking — 불가능/절대금지)
| ID | 규칙 | 판정 |
|---|---|---|
| H1 | 과적 금지 | 총질량 ≤ **7 kg(목업)** |
| H2 | 적재함(62×21×27cm) 내부 | BBox 경계 내 (장척 파이프 오버행만 예외) |
| H3 | 화물 겹침 금지 | 같은 층 AABB 중첩 X |
| H4 | 파이프 x축(주행방향) 배치 | 실제 운송 기준 |
| H5 | 파이프 바닥층 적재 | 구름 방지 |
| H6 | 파이프 위·아래 타화물 금지 | 접촉면 부족 |
| H7 | 포대 위 상자 금지 (포대 위엔 포대만) | 포대 변형 방지 |
| H8 | 적재 높이 27cm 초과 금지 | 적재함 높이 한도 |

### 4.2 Hybrid (최소선 = Hard, 그 위 = Soft) — 학습 유연성 위해 완화
| 항목 | Hard(금지) | Soft(가점) |
|---|---|---|
| 지지율 | **< 50% 금지** (붕괴 방지) | 50~100% 높을수록 + _(문서 원안은 70% 통 Hard)_ |
| CoG 편중 | **±15% 초과 금지** | 그 안에서 중앙 수렴할수록 + |

> ⚠️ Hard가 너무 많으면 초반 유효 행동이 없어 학습이 안 붙음 → **진짜 불가능한 것만 Hard**, 나머지는 Soft/Hybrid.

### 4.3 Soft Constraint = Reward
→ 5절(보상 함수)로 통합. (좌우/전후 하중편차·축하중은 **Reward에서 제외** — CoG항과 중복, 동적 검증에서 실측)

---

## 5. 보상 함수 (Reward) — 3 목적함수 가중합 (문서 3.5)

```
R = w_LE·LoadingEfficiency + w_CGS·CoG_Stability + w_SS·StackingStability
초기: w_LE=0.50, w_CGS=0.40, w_SS=0.10
```
| 목적함수 | 초기 W | 구성요소 |
|---|---|---|
| **Loading Efficiency** | 0.50 | 부피활용율 / Dead Space / Grid 밀집 / 벽·화물 접촉 |
| **CoG Stability** ⭐핵심 | 0.40 | CoG_x·y 중앙 수렴, **CoG_z(높이) 낮게** |
| **Stacking Stability** | 0.10 | 무거운 것 아래 / 상단 평탄도 / 포대 네스팅 |

- **가중치는 임의 X → DOE 민감도 분석으로 결정**
  `Representative DOE → Sensitivity → Weight 선정 → RL → 동적 검증`
- **제외됨**: 좌우/전후 하중편차·축하중 (CoG항과 중복 → 동적 Validation에서 실측)
- 스텝 보상: 배치마다 CoG 중앙·밀집을 소폭 shaping. 최종 보상: 위 R 합산.
- Hard 위반은 마스킹으로 안 나오지만, 새어나오면 큰 음수 + 종료.

### 5.4 2차 보상 — 동적 예측모델 (정적 다음 단계, 확정된 로드맵)
- 2000 케이스 동적 데이터로 "배치특징 → 전복확률/최대LTR/위험등급" 예측기 학습
- `R_final`의 안정 항을 예측기 출력으로 교체 → **현실적 위험** 반영
- 전제: 데이터가 안전~전복 스펙트럼을 담아야 함 (#4 해결 + 위험 유도 필요)

## 6. 커리큘럼 (난이도 점증)

1. 화물 3~4개, 넉넉한 공간 → 배치 자체 학습
2. 화물 수 증가·공간 빡빡 → 밀집 학습
3. 무거운 화물·편심 유혹 상황 → 균형 학습
4. payload 근처 → 과적 회피 + 안정 동시

## 7. 학습 · 검증

- **학습**: `mlagents-learn config.yaml` (PPO). 에피소드마다 화물 목록 랜덤.
- **평가 지표**: 규제 위반율↓, 평균 StabilityMargin↑, |LTR|↓, 공간활용↑
- **검증(루프 닫기)**: 학습된 에이전트 배치 → 케이스 JSON 저장 → **동적 주행**으로 실제 roll/LTR/전복 측정 → "RL vs 랜덤/휴리스틱" 비교

## 9. 결정사항

**확정됨:**
- ✅ **화물 순서**: 에이전트가 선택 (item 선택을 행동에 포함)
- ✅ **보상 소스**: 정적만 먼저(1차) → 이후 동적 예측기(2차)
- ✅ **payload**: 사용자 설정 파라미터 (목업 kg, 실차 = ×massScale)
- ✅ **고정 여부**: RL 변수 아님 — 전부 고정(secured) 가정
- ✅ **규제 목록**: 수령 완료 → 4절에 Hard/Hybrid, 5절에 Soft(Reward) 반영
- ✅ **적재함**: 62 × 21 × 27 cm (높이 한도 27cm)
- ✅ **payload**: 7 kg 목업 (= 700 kg 실차)
- ✅ **하중편차·축하중**: Reward 제외 → 동적 Validation에서 실측

- ✅ **최적화 목표**: **안정성**(전복 위험 최소) — 패킹 밀도 아님
- ✅ **격자 해상도**: **4cm(96셀)** 로 확정.
  - 근거: 목표가 안정성이므로 배치 위치 ±2cm 오차는 CoG/LTR에 거의 영향 없음 → 1cm/연속 배치 불필요.
  - 화물 치수가 어중간해도(10.4·7.9cm) mm 정밀은 안정성에 무의미. 행동 공간이 작아 학습도 빠름.
  - 나중에 필요하면 2cm로 상향 가능(파라미터 교체). 관측 점유맵도 4cm(16×6) 기준.

**남은 결정:**
1. **정적→동적 전환 시점**: 1차 학습이 어느 성능에 도달하면 2차로 갈지
   (동적 보상은 #4 해결 + 위험 데이터 확보가 전제)
